# codewithrafiq.github.io

Personal single-page portfolio of **MD Rafiqul Islam** — Software Engineer.
Built with **React 19** and deployed to **GitHub Pages**.

🔗 Live: https://codewithrafiq.github.io

## Repository layout

```
.
├── app/            # React source (all development files live here)
│   ├── src/        #   components & styles (App.js, App.css, …)
│   ├── public/     #   static assets + index.html template
│   ├── scripts/    #   copy-to-root.js (publishes build to repo root)
│   └── package.json
│
├── index.html      # ← compiled site (GitHub Pages serves from the ROOT)
├── static/         # ← compiled JS/CSS bundles
├── profile.jpeg    # ← other compiled assets
└── …               #   (all generated by the build, committed for Pages)
```

GitHub Pages for a user site (`<user>.github.io`) serves from the **root** of the
deployed branch, so the production build is emitted to the repository root while
the React source stays neatly inside `app/`.

## Develop

```bash
cd app
npm install      # first time only
npm start        # dev server at http://localhost:3000
```

## Build & deploy

```bash
cd app
npm run build    # compiles, then copies the build to the repo root
```

`npm run build` runs `react-scripts build` and then `scripts/copy-to-root.js`,
which cleans the previous generated files at the root and copies the fresh build
up so `index.html` lands where GitHub Pages expects it.

Then commit and push the root files to the branch configured in
**Settings → Pages** (typically `main` / root) and the site goes live.
